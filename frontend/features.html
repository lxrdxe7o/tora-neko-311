Kyber-512 Key Encapsulation Lattice-based quantum-safe encryption
NIST FIPS 203 ML-KEM 512-bit security

Technical Overview

Kyber is a key encapsulation mechanism (KEM) based on the Module Learning With Errors (Module-LWE) problem. Unlike RSA and elliptic curve cryptography which are vulnerable to Shor's algorithm on quantum computers, Kyber's security relies on lattice-based mathematical problems that remain computationally hard even for quantum adversaries.

The algorithm was selected by NIST in 2022 and standardized as FIPS 203 (ML-KEM) in 2024, representing years of cryptanalysis and peer review by the global cryptographic community.

How We Implement It

  • Passport Encryption: When you enter your passport number, we generate a fresh Kyber keypair on the server
  • Key Encapsulation: A shared secret is derived using the Kyber KEM protocol
  • Hybrid Encryption: The shared secret is used as a key for AES-256-GCM symmetric encryption
  • Storage: Only the ciphertext and public key are stored in the database
  • Decryption: Retrieval requires the private key held securely on the server

Security Parameters

  • Security Level: NIST Level 1 (comparable to AES-128 for quantum attacks)
  • Public Key Size: 800 bytes
  • Ciphertext Size: 768 bytes
  • Security Notion: IND-CCA2 secure

Why It Matters

"Harvest Now, Decrypt Later" is a real threat where adversaries collect encrypted data today with the intention of decrypting it once quantum computers become available. By implementing Kyber-512 today, we ensure your data remains confidential even in a post-quantum world.

Dilithium3 Digital Signatures Lattice-based quantum-resistant signatures
NIST FIPS 204 ML-DSA Security Level 3

Technical Overview

Dilithium is a digital signature scheme based on lattice problems (SIS and LWE). Standardized as NIST FIPS 204, it provides strong security guarantees against both classical and quantum adversaries.

How We Implement It

  • Booking Confirmation: Upon successful seat reservation, a booking payload is created
  • Signature Generation: The server signs the payload using a Dilithium3 private key
  • Verification: Anyone can verify the signature using the public key
  • Tamper Detection: Any modification to booking data invalidates the signature

Why It Matters

Traditional signature schemes (RSA, ECDSA) are vulnerable to quantum attacks. Dilithium ensures that even with a quantum computer, signatures cannot be forged, guaranteeing ticket authenticity.

Quantum Random Number Generation Hadamard gate simulation for true entropy
QRNG 128-bit entropy

Technical Overview

Our implementation simulates quantum behavior using Hadamard gates applied to qubits in superposition states, producing high-quality entropy for booking reference generation.

How We Implement It

  • Qubit Initialization: Start with |0⟩ state
  • Hadamard Transform: Apply H gate to create superposition
  • Measurement: Collapse to |0⟩ or |1⟩ with equal probability
  • Booking Reference: Encode entropy as QREF-XXXX-XXXX-XXXX format

Why It Matters

Traditional booking IDs are often predictable. Quantum-inspired random IDs ensure unpredictability, preventing brute-force enumeration and protecting privacy.
